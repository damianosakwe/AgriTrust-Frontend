Implements a comprehensive transaction recovery system that prevents double
submissions on the Soroban network when users experience browser crashes,
tab refreshes, or network interruptions during escrow deposits.

## Key Features

- Transaction state persistence using sessionStorage (survives refresh)
- Automatic recovery queue on app initialization
- 5-second recovery timeout with blockchain status check
- LRU eviction for max 100 tracked transactions
- Professional UI recovery banner with retry/dismiss actions
- Full lifecycle tracking: preparing → broadcasting → pending → confirmed/failed
